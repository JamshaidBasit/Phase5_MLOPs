Jab hum code likhte hain, toh hum Git use karte hain taake purana code recover kar sakein. Lekin Git bari files (datasets, model weights) ke liye nahi bana. Agar aap 1GB ka data Git par push karoge, toh Git ro dega. DVC isi maslay ka hal hai.

1. Core Concept: DVC Kaise Kaam Karta Hai?
DVC asli data ko Git mein nahi dalta. Wo asli data ko aapke local computer ya Cloud (S3, Google Drive, Azure) par rakhta hai aur Git mein sirf ek choti si metadata file (.dvc) dalta hai.

Code: Git sambhalta hai.

Data & Models: DVC sambhalta hai.

Connection: Git ki .dvc file batati hai ke asli data DVC ke storage mein kahan para hai.

===============================================================================================
2. Step-by-Step Practical Workflow (Coding Example)
Maun lo aapke paas ek folder hai data/ jisme large_dataset.csv para hai.

Step A: Initialize DVC
Pehle Git init karo, phir DVC.

Bash
git init
dvc init
git commit -m "Initialize DVC"

=======================================================
Step B: Data ko DVC ke hawale karna
Ab hum file ko DVC mein add karenge.

=========================================================

Kya hua? * DVC ne asli file ko .gitignore mein daal diya (taake Git usay touch na kare).

DVC ne ek nayi file banayi: data/large_dataset.csv.dvc. Ye choti si text file hai.
========================================================


Step C: Git ko batana
Ab hum sirf .dvc file ko Git mein commit karenge.

git add data/large_dataset.csv.dvc data/.gitignore
git commit -m "Add data tracking"

=================================================

3. Remote Storage (The Real Power)
Ab aap apna data Cloud par save kar sakte hain taake aapka team member bhi use kar sake.

Bash
# Google Drive ya S3 ko remote set karna
dvc remote add -d myremote gdrive://your_folder_id_here

# Data ko cloud par bhejna
dvc push
Ab jab aapka dost code clone karega, wo sirf ye likhega:

Bash
git pull
dvc pull
Aur Boom! 1GB ka data uske paas khud-ba-khud aa jayega.

=============================================================
4. Data Versioning (Switching between data)
Maun lo aapne naya data add kiya aur model kharab ho gaya. Aap purane data par wapas jana chahte ho:

Bash
# Purane git commit par jao jahan data purana tha
git checkout <old_commit_id>

# DVC ko kaho ke file waisi kar de jaisi us waqt thi
dvc checkout
